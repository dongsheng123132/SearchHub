# 🔒 Chrome 安全警告解决方案

## 问题说明

当您尝试安装 SearchHub 扩展时，Chrome 显示以下警告：

```
继续操作有风险，请三思
增强型安全浏览功能不信任此扩展程序

该程序可以:
- 读取和更改您在一些网站上的数据
- 读取您的浏览记录
- 显示通知
- 查看和管理您的标签页组
```

## 为什么会出现这个警告？

这是 **Chrome 的正常安全机制**，不是错误！

### 原因分析

1. **权限声明**：扩展需要某些权限才能正常工作
2. **新扩展**：未上架的扩展会显示更严格的警告
3. **权限数量**：Chrome 会警告所有需要 `tabs`、`storage` 等权限的扩展

### 为什么 SeekAll AI 没有警告？

可能的原因：
- ✅ **已上架并通过审核**：已审核的扩展警告较少
- ✅ **使用不同的权限策略**：可能使用了 optional_permissions
- ✅ **权限更少**：可能功能更简单，需要的权限更少

## ✅ 已优化的权限配置

我已经优化了 `manifest.json`，**大幅减少了权限**：

### 优化前
- ❌ `host_permissions`: **118个域名**
- ❌ 包含所有搜索引擎域名（实际上不需要）

### 优化后
- ✅ `host_permissions`: **仅14个域名**
- ✅ 只保留 content_scripts 需要的域名（用于注入"继续对话"功能栏）
- ✅ **移除了所有搜索引擎域名**（打开标签页不需要权限）

### 关键发现

**打开标签页不需要 `host_permissions`！**

`host_permissions` 只在以下情况需要：
- ✅ 注入 content_scripts（我们已经优化，只保留必要的）
- ❌ 打开标签页（不需要权限）

## 📋 当前权限说明

### 必需的权限（无法移除）

| 权限 | 用途 | 为什么需要 |
|------|------|-----------|
| `activeTab` | 响应用户操作 | 用户点击扩展图标时访问当前标签页 |
| `tabs` | 创建标签页 | 打开多个搜索引擎标签页 |
| `tabGroups` | 分组标签页 | 将搜索结果自动分组 |
| `sidePanel` | 显示侧边栏 | 显示垂直标签列表 |
| `windows` | 管理窗口 | 支持"在新窗口打开"功能 |
| `storage` | 保存设置 | 保存用户选择的引擎、组合等 |
| `contextMenus` | 右键菜单 | "搜索选中文本"功能 |
| `scripting` | 注入脚本 | 在AI聊天页面注入功能栏 |
| `notifications` | 显示通知 | 搜索完成时通知（可选） |
| `system.display` | 显示器信息 | 新窗口打开时选择合适位置 |

## 🎯 如何减少警告？

### 1. 优化权限（✅ 已完成）

已从 118 个域名减少到 14 个域名。

### 2. 添加隐私政策（⚠️ 需要您完成）

在 `manifest.json` 中，我已经添加了 `privacy_policy` 字段，但您需要：

1. **发布隐私政策**：
   - 将 `PRIVACY_POLICY.md` 的内容发布到 GitHub Gist
   - 或发布到您的网站

2. **更新 manifest.json**：
   ```json
   "privacy_policy": "https://gist.github.com/yourusername/your-gist-id"
   ```
   替换为您的实际URL

### 3. 在商店描述中说明权限用途

在 Chrome Web Store 提交时，在描述中说明：
- 为什么需要 `tabs` 权限（打开多个搜索引擎）
- 为什么需要 `storage` 权限（保存用户设置）
- 为什么需要 `host_permissions`（只在AI网站注入功能栏）

### 4. 通过 Chrome 审核

审核通过后，警告会显著减少。

## ⚠️ 重要提示

- 这是 Chrome 的安全机制，保护用户隐私
